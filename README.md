## 1. 執行摘要

### 1.1 產品願景

Project Highlight v1.0 是一套 **AI 驅動的個人運動高光捕捉系統**，專為「個人時代」下重視社交媒體形象的運動愛好者而設計。我們不僅僅是一台攝影機，更是一個**將每場運動轉化為社交媒體內容的智能引擎**。

### 1.2 時代背景：從「群體時代」到「個人時代」

> **社交媒體帶來的根本性改變:** 過去是「群體時代」——人們追求融入團體、統一行動。如今是「個人時代」——每個人都是自己的品牌、自己的媒體。

> **體育界尤為明顯:** 運動員和運動愛好者比任何群體都更重視自己在社交媒體上的形象。一個漂亮的扣籃、一次完美的射門、一記絕殺三分——這些不只是運動瞬間，更是**社交貨幣**。

> **我們的解決方案:** AI 自動捕捉用戶在運動中的每一個最佳動作。每一場運動，都是用戶經營社交媒體的機會。你只需要專注運動，其餘的交給我們。

### 1.3 使命宣言

> "讓每位運動愛好者都擁有專屬 AI 攝影師，將每一場運動轉化為個人品牌的高光時刻，在個人時代中脫穎而出。"

### 1.4 產品定位

**我們是什麼:** 一個 AI 驅動的個人運動高光自動捕捉與社交媒體內容生成系統  
**我們不是什麼:**  
- ❌ 專業體育轉播系統（我們不做專業賽事直播）
- ❌ 普通運動攝影機（GoPro 需要你手動操作）
- ❌ 影片編輯軟件（我們不是 Premiere 或 CapCut）
- ❌ 社交媒體平台（我們不是 Instagram 或 TikTok）

**類比:** 就像 Tesla Autopilot 解放了你的雙手 —— 我們的 AI 攝影師解放了你的注意力。**你專注打球，AI 幫你捕捉高光。**

**核心原則:**
> "每一場運動，都是一次社交媒體的機會。你負責表演，AI 負責記錄。"

### 1.5 硬體三件套

```
Project Highlight 硬體架構：

1. 📹 拍片機 (AI Camera Rig)
   - 多機位高清攝影裝置
   - 自動追蹤運動中的人物
   - 即時拍攝高幀率影片

2. 💾 基本 NAS (Network Attached Storage)
   - 本地影片存儲
   - 高速讀寫，支持多路同時寫入
   - 自動備份與影片管理

3. 🧠 LOCAL AI (M4 Mac Mini)
   - 本地端 AI 推理運算
   - 動作識別與高光偵測
   - 影片裁剪與後期處理
   - 無需雲端，保護用戶隱私
```

## 4. 功能規格

### 4.1 核心功能模組

#### 4.1.1 AI 智能捕捉引擎 📹

**核心邏輯:** AI 自動識別運動中的「高光動作」並即時捕捉

**動作識別能力（v1.0 優先支持）:**

| 運動類型 | 可識別動作 | 識別方式 | 優先級 |
|---------|-----------|---------|-------|
| 🏀 籃球 | 進球、扣籃、蓋帽、搶斷 | 球體軌跡追蹤 + 籃框震動感測 | P0 |
| ⚽ 足球 | 進球、精彩過人、遠射 | 球網震動偵測 + 球體軌跡 | P1 |
| 🎾 網球/羽球 | 扣殺、致勝球、精彩回擊 | 揮拍動作識別 + 球速偵測 | P2 |

**捕捉流程:**
```
運動進行中：
1. 多機位同步拍攝（連續錄影）
2. M4 Mac Mini 即時運行 AI 動作識別
3. 偵測到高光動作 → 標記時間戳
4. 自動裁剪前後 10-30 秒片段
5. 選取最佳機位角度
6. 生成高光影片 → 存入 NAS
7. 推送通知至用戶手機

用戶收到通知：
8. 預覽高光片段
9. 選擇濾鏡/音樂/文字
10. 一鍵分享至 IG/TikTok/YouTube Shorts
```

**AI 捕捉決策邏輯:**
```python
class HighlightDetector:
    """AI 高光偵測引擎"""
    
    def __init__(self):
        self.action_models = {
            'basketball': BasketballActionModel(),
            'football': FootballActionModel(),
            'tennis': TennisActionModel()
        }
        self.highlight_threshold = 0.75  # 高光信心度閾值
    
    def detect_highlight(self, video_frame, sport_type, sensor_data):
        """
        即時偵測高光動作
        
        Args:
            video_frame: 當前影片幀
            sport_type: 運動類型
            sensor_data: 感測器數據（籃框震動、球網偵測等）
        
        Returns:
            {
                'is_highlight': bool,
                'confidence': float,    # 0-1
                'action_type': str,     # 'goal', 'dunk', 'save' 等
                'timestamp': datetime,
                'camera_angles': list,  # 推薦機位
                'clip_range': tuple     # (start_sec, end_sec)
            }
        """
        model = self.action_models.get(sport_type)
        if not model:
            return {'is_highlight': False}
        
        # 結合視覺識別與感測器數據
        visual_score = model.analyze_frame(video_frame)
        sensor_score = model.analyze_sensor(sensor_data)
        
        # 加權融合
        combined_score = visual_score * 0.6 + sensor_score * 0.4
        
        if combined_score >= self.highlight_threshold:
            return {
                'is_highlight': True,
                'confidence': combined_score,
                'action_type': model.classify_action(video_frame),
                'timestamp': datetime.now(),
                'camera_angles': self.rank_camera_angles(video_frame),
                'clip_range': self.calculate_clip_range(combined_score)
            }
        
        return {'is_highlight': False, 'confidence': combined_score}
```

#### 4.1.2 本地 AI 運算引擎 (M4 Mac Mini) 🧠

**核心邏輯:** 所有 AI 運算在本地完成，無需上傳雲端，保護用戶隱私

**M4 Mac Mini 運算架構:**
```
M4 Mac Mini 本地 AI Pipeline：

┌─────────────────────────────────────────────────────┐
│  📹 多機位輸入 (Camera Feeds)                        │
│  → 2-4 路 1080p/60fps 影片串流                       │
├─────────────────────────────────────────────────────┤
│  🧠 AI 推理引擎 (Neural Engine)                      │
│  → 動作偵測模型 (Action Detection)                    │
│  → 人物追蹤模型 (Person Tracking)                     │
│  → 物體識別模型 (Object Recognition: 球/框/網)        │
├─────────────────────────────────────────────────────┤
│  ✂️ 智能裁剪引擎 (Smart Clip Engine)                  │
│  → 自動選取最佳時間範圍                              │
│  → 自動選取最佳機位角度                              │
│  → 自動添加慢動作回放                                │
├─────────────────────────────────────────────────────┤
│  💾 存入 NAS → 📱 推送通知至用戶 App                 │
└─────────────────────────────────────────────────────┘
```

**為什麼選擇 M4 Mac Mini？**
```
M4 Mac Mini 的優勢：

1. Apple Neural Engine
   - 38 TOPS（每秒 38 萬億次運算）
   - 專為 AI/ML 推理優化
   - 低功耗高效能

2. 統一記憶體架構
   - 16/24GB 統一記憶體
   - GPU 與 Neural Engine 共享記憶體
   - 多路影片串流無壓力

3. 本地運算的優勢
   - 零網路延遲（關鍵！）
   - 用戶數據不離開場地
   - 無需支付雲端 GPU 費用
   - 即使斷網也能正常運作

4. 成本效益
   - 一次性購入 ~$600-800 USD
   - vs 雲端 GPU：$0.5-3/小時
   - 6 個月即回本
```

#### 4.1.3 NAS 存儲系統 💾

**核心邏輯:** 高速本地存儲，確保多路影片同時寫入不掉幀

**NAS 架構:**
```
基本 NAS 配置：

硬體規格：
- 2-Bay NAS（如 Synology DS224+）
- 2 × 4TB HDD (RAID 1 鏡像)
- 有效存儲: 4TB
- 10GbE 網路連接

存儲策略：
- 原始影片：保留 7 天後自動清理
- 高光片段：永久保留（用戶可手動刪除）
- AI 決策日誌：永久保留
- 預計每場運動原始影片：~50GB
- 預計每場高光片段：~2GB

存儲容量估算：
- 4TB 有效空間
- 每天 2 場運動 = ~104GB 原始 + ~4GB 高光
- 7 天原始影片循環 = ~728GB
- 高光片段累積：每月 ~60GB
- 預計可用 > 12 個月才需擴展
```

## 10. 風險管理

### 10.1 技術風險

**風險 1: AI 動作識別準確率不足**
- **可能性:** 中高（初期模型需要大量訓練數據）
- **影響:** 高（核心產品體驗）
- **緩解:**
  - 初期聚焦籃球進球（最容易識別的動作）
  - 結合感測器數據（籃框震動）降低純視覺依賴
  - 持續收集數據優化模型
  - Lawrence 提供 AI 演算法諮詢

**風險 2: M4 Mac Mini 運算能力瓶頸**
- **可能性:** 低中（M4 Neural Engine 性能強勁）
- **影響:** 中（影響延遲和處理量）
- **緩解:**
  - 優化模型量化（INT8/FP16）
  - 優先處理高信心度動作
  - 如需擴展可升級至 M4 Pro/Max

**風險 3: 多機位同步困難**
- **可能性:** 中（時間同步是常見挑戰）
- **影響:** 中（影響多角度高光品質）
- **緩解:**
  - 使用 NTP 時間同步
  - 音頻同步作為備份方案
  - 初期從 2 機位起步

### 10.2 商業風險

**風險 4: 球場合作意願低**
- **可能性:** 中（新概念需要市場教育）
- **影響:** 高（B2B 是核心收入來源）
- **緩解:**
  - 提供免費試用期（1-3 個月）
  - 用數據證明吸引客流的效果
  - 從 1 個球場做出標竿案例

**風險 5: 用戶隱私顧慮**
- **可能性:** 中高（攝影機在公共場所引發擔憂）
- **影響:** 高（可能阻礙推廣）
- **緩解:**
  - 所有 AI 運算在本地完成（不上傳雲端）
  - 清晰的隱私政策和告示牌
  - 用戶必須主動註冊才會被 AI 追蹤
  - 非註冊人員面部自動模糊化

### 10.3 營運風險

**風險 6: 團隊招募困難**
- **可能性:** 中（需要特定技能組合）
- **影響:** 中高（影響開發進度）
- **緩解:**
  - 允許遠端/兼職/合約制
  - 初期角色可重疊（Backend + QA）
  - 必要時考慮外包部分開發

**風險 7: 硬體成本超預算**
- **可能性:** 低中
- **影響:** 中（影響每場地部署成本）
- **緩解:**
  - 先從最基本配置開始（2 機位 + 基本 NAS + M4 Mac Mini）
  - 制定詳細的硬體 BOM (Bill of Materials)
  - 尋求量購折扣
